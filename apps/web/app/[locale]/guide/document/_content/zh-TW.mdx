# PDF 文件翻譯

上傳任意 PDF，獲得完整的雙語輸出——並排 HTML 檢視與交替排版的 Markdown，由與文字翻譯相同的 AI 模型驅動。

---

## 適用情境

- **學術論文** — 翻譯研究文章，同時保留原有結構
- **合約與法律文件** — 審閱外文協議
- **技術文件** — 手冊、規格書、資料表
- **書籍與長篇報告** — 多頁文件一次完成翻譯

---

## 處理流程

1. **上傳** — 拖曳 PDF 檔案或點擊選擇檔案
2. **擷取** — 伺服器解析 PDF 並按頁擷取文字
3. **分段** — 將文字拆分為適合翻譯的片段
4. **翻譯** — 每個片段傳送至您所選的 AI 模型
5. **輸出** — 結果組裝為雙語 HTML 與 Markdown

## 輸出格式

**並排 HTML** — 原文與譯文分兩欄展示，按頁呈現。可下載為獨立 HTML 檔案，離線查看。

**交替 Markdown** — 原文段落後緊接其譯文，如此循環貫穿全文。適合貼入筆記應用程式或進一步編輯。

## 已知限制

- **掃描版 PDF**（純圖片，無文字層）目前不支援。擷取器需要可選取的文字。OCR 支援計畫於未來版本推出。
- 表格與複雜排版在輸出中可能會遺失部分格式。
- 非常大的檔案可能需要數分鐘處理。

## 常見問題

**文件翻譯支援哪些 AI 模型？**
與文字翻譯相同的模型。免費使用者可使用 Google、Microsoft、LibreTranslate 和 GPT-4o mini；Pro 使用者可存取全部 11 個模型。

**翻譯需要多長時間？**
通常一份 10 頁的文件需要 1–3 分鐘，具體取決於所選模型與伺服器負載。您可以關閉頁面稍後回來查看——結果已儲存於您的工作階段中。

**可以翻譯 URL 連結的 PDF 嗎？**
可以。在 `/document` 頁面 URL 後加上 `?src=<連結>` 參數，系統會自動擷取並翻譯。

**PDF 檔案會被永久儲存嗎？**
來源檔案處理完成後即刪除。輸出的 HTML / MD 檔案依據您所在方案的保留政策保存（請參閱上方配額規則）。
