# PDF 文档翻译

上传任意 PDF，获得完整的双语输出——并排 HTML 视图与交替排版的 Markdown，由与文本翻译相同的 AI 模型驱动。

---

## 适用场景

- **学术论文** — 翻译研究文章，同时保留原有结构
- **合同与法律文件** — 审阅外文协议
- **技术文档** — 手册、规格书、数据手册
- **书籍与长报告** — 多页文档一次性完成翻译

---

## 处理流程

1. **上传** — 拖拽 PDF 文件或点击选择文件
2. **提取** — 服务器解析 PDF 并按页提取文字
3. **分段** — 将文字拆分为适合翻译的片段
4. **翻译** — 每个片段发送至你所选的 AI 模型
5. **输出** — 结果组装为双语 HTML 和 Markdown

## 输出格式

**并排 HTML** — 原文与译文分两列展示，按页呈现。可下载为独立 HTML 文件，离线查看。

**交替 Markdown** — 原文段落后紧跟其译文，如此循环贯穿全文。适合粘贴到笔记应用或进一步编辑。

## 已知限制

- **扫描版 PDF**（纯图片，无文字层）暂不支持。提取器需要可选中的文本。OCR 支持计划在未来版本中推出。
- 表格和复杂排版在输出中可能会丢失部分格式。
- 非常大的文件可能需要数分钟处理。

## 常见问题

**文档翻译支持哪些 AI 模型？**
与文本翻译相同的模型。免费用户可使用谷歌、微软、LibreTranslate 和 GPT-4o mini；Pro 用户可访问全部 11 个模型。

**翻译需要多长时间？**
通常一份 10 页的文档需要 1–3 分钟，具体取决于所选模型和服务器负载。你可以关闭页面稍后回来查看——结果已存储在你的会话中。

**可以翻译 URL 链接的 PDF 吗？**
可以。在 `/document` 页面 URL 后加上 `?src=<链接>` 参数，系统会自动获取并翻译。

**PDF 文件会被永久存储吗？**
源文件处理完成后即删除。输出的 HTML / MD 文件根据你所在档位的保留策略保存（见上方配额规则）。
