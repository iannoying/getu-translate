> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans.
> **Parent design:** `docs/plans/2026-04-22-m4-video-subtitles-design.md`
> **Predecessor:** M4.1 (#108) — Bilibili adapter

**Goal:** Add TED talks (ted.com/talks/{slug}) subtitle translation. TED is the simplest new platform — static pages (no SPA), public transcript API (no auth), HTML5 `<video>` element.

**Architecture:**
- `TedSubtitlesFetcher` queries `https://www.ted.com/talks/{slug}/transcript.json` (public, no cookies)
- `platforms/ted/config.ts` targets TED's own player DOM (`.ted-player__video-player` / equivalent; may need DOM inspection)
- `platforms/ted/handler.ts` registers with `PlatformRegistry`
- `init-ted-subtitles.ts` handles the simple case: no SPA — page load is full navigation
- manifest adds `*://*.ted.com/*` + `*://www.ted.com/*`

**API research (TED's transcript endpoints):**

TED's public endpoint pattern:
- `https://www.ted.com/talks/{slug}/transcript.json?language={lang}` — returns `{ paragraphs: [{ cues: [{ time: 1234, text: "..." }] }] }`
- Alternative: `https://pa.tedcdn.com/api/transcripts/{id}.json` (CDN)
- Fallback: parse `<track kind="captions" src="...">` from the `<video>` element directly

**Shape:**

```ts

  kind: "ted" = "ted"

  async fetchTracks(videoId: { slug: string }): Promise<RawTrack[]> {
    // Query /talks/{slug}/transcript.json with ?language=en (default)
    // Response also includes available languages via data.talks[0].downloads[].subtitled_downloads
    // Return RawTrack[] with language codes as iso
  }

  async fetchCues(track: RawTrack): Promise<SubtitleCue[]> {
    // Fetch transcript JSON, flatten paragraphs.cues[].time + text
    // TED's `time` is milliseconds from video start
  }

  shouldUseSameTrack(prev, next) { return prev?.trackId === next?.trackId }
}
```

**Quirks:**
- TED paragraph boundaries are coarse — each paragraph has ~3-8 cues; fetcher flattens to a flat `SubtitleCue[]`
- Transcript endpoint may 404 for some talks (no transcript available); graceful: return empty tracks, UI shows "no subtitles available"
- Some talks have multiple speakers; transcript JSON includes `speaker` field — ignore for M4.2 (reserve for M5/M6 enhancements)

**Simpler than Bilibili:** TED pages are mostly full page loads. Still handle:
- URL pattern: `/\/talks\/[^/]+/` (reject non-talk pages)
- Watch for slug change via `beforeunload` or a lightweight URL poll (sub-second cost)
- In most cases, just initialize once on page load

**Shape:**

```ts
export function initTedSubtitles() {
  const tryInit = async () => {
    const slug = getTedSlug()
    if (!slug) return
    // ... mount + initialize adapter (mirrors youtube pattern)
  }
  void tryInit()
  window.addEventListener("popstate", () => void tryInit())  // back/forward
}
```

## Risks

| Risk | Mitigation |
|------|------------|
| `/transcript.json` endpoint removed by TED | Fallback: parse `<track>` VTT from `<video>` element |
| TED React class names change | Use stable attributes (`data-testid`) where possible; tests for selector robustness in adapter |
| Non-English transcript languages | First pass: only English; future task adds multi-language support |
| Embedded TED player on other sites (`embed.ted.com`) | M4.2 only matches `*.ted.com/*`; embeds out of scope |

## Out of scope

- Non-English source languages (M4.3+)
- TED ED lessons (different URL pattern)
- Conference archives with special URL structure
- Multi-speaker annotation
