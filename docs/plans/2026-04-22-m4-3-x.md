> **Predecessor:** M4.2 (#109) — TED adapter. Final PR of M4 stack.

**Goal:** Add X (Twitter) video subtitle translation. Most X videos are short (< 2:20) and often have no native captions — but when captions exist, they come from the video element's `textTracks` (VTT) or from an HLS manifest. This PR targets the simpler `video.textTracks` path.

**Architecture:**
- `XSubtitlesFetcher` reads cues directly from `HTMLVideoElement.textTracks[0].cues` rather than fetching from a network API (X doesn't expose public caption endpoints)
- `platforms/x/config.ts` targets tweet video containers (`article[data-testid="tweet"] video`)
- `platforms/x/handler.ts` matches both `twitter.com` and `x.com`
- SPA navigation via URL polling (X's routing is complex; no clean events)
- manifest adds `*://twitter.com/*` + `*://x.com/*`

**Out of scope:**
- HLS manifest parsing (for videos where captions come from `.m3u8`, not `textTracks`)
- Twitter Spaces (audio-only)
- Grok video / other non-standard embeds
- Live streams

## Task 1: X subtitle fetcher (textTracks-based, not network)

**Files:**
- Create `apps/extension/src/utils/subtitles/fetchers/x/index.ts`

**Unlike YouTube/Bilibili/TED, X has no public caption API.** Extract captions from the DOM video element:

```ts
export class XSubtitlesFetcher implements SubtitlesFetcher {
  kind: "x" = "x"

  async fetchTracks(videoId: { tweetId: string }): Promise<RawTrack[]> {
    const video = document.querySelector<HTMLVideoElement>(X_VIDEO_SELECTOR)
    if (!video) return []
    const tracks = Array.from(video.textTracks ?? [])
    return tracks
      .filter(t => t.kind === "captions" || t.kind === "subtitles")
      .map(t => ({ trackId: t.language || t.label, language: t.language, label: t.label, source: t }))
  }

  async fetchCues(track: RawTrack): Promise<SubtitleCue[]> {
    const tt = track.source as TextTrack
    tt.mode = "hidden"  // ensure cues load without showing native captions
    const cues = Array.from(tt.cues ?? []) as VTTCue[]
    return cues.map(c => ({ startTime: c.startTime * 1000, endTime: c.endTime * 1000, text: c.text }))
  }

  shouldUseSameTrack(prev, next) { return prev?.trackId === next?.trackId }
}
```

**Quirks:**
- X lazy-loads video metadata — `textTracks` may be empty on first read. Adapter's existing retry-with-delay should handle; if not, add an `await waitForTextTracks()` helper
- Most X videos have zero textTracks — fetcher returns empty array gracefully; UI shows "no captions available"
- `VTTCue` type isn't always in TypeScript DOM lib — cast via structural subset if needed

**Commit:** `feat(subtitles): add XSubtitlesFetcher (M4.3 Task 1)`

## Task 3: X SPA navigation + init

**Files:**
- Create `apps/extension/src/entrypoints/subtitles.content/init-x-subtitles.ts`

X uses React Router; navigation is all pushState-based (same as Bilibili). Reuse the `history.pushState` monkey-patch pattern from `init-bilibili-subtitles.ts`:

```ts
patchHistoryForSpaNavigation()
window.addEventListener("bilibili:spa-navigated", () => void tryInit())  // reuse event or rename
window.addEventListener("popstate", () => void tryInit())
```

Actually since both platforms use the same pattern, **extract `patchHistoryForSpaNavigation()` into `utils/dom/history-patch.ts`** (refactor) and emit a shared event `getu:spa-navigated`. Both Bilibili init and X init listen for it.

Keep the refactor minimal: if the patch + event is already emitted by one platform, the other just needs to subscribe. Check M4.1's `init-bilibili-subtitles.ts` after rebase — if the monkey-patch is idempotent, no refactor needed.

URL pattern: status pages only. Not every tweet has a video — early return if no video found.

## Risks

| Risk | Mitigation |
|------|------------|
| `textTracks.cues` is lazy — empty on first read | Retry with timeout (adapter already has this pattern); poll up to 3s |
| X React re-renders destroy video element mid-playback | Adapter re-mounts on SPA nav; if video element swap mid-page, next `textlayerrendered`-equivalent would catch it (but X may need its own listener) |
| `data-testid` selectors change between X releases | Flag uncertain selectors with `TO VERIFY` comments; manual smoke before merge |
| twitter.com and x.com both need to work | Test both domains in fetcher tests + registry dispatch |

- HLS manifest VTT parsing (for captions from `.m3u8`)
- Twitter Spaces (audio)

- Timeline auto-play videos (only status pages)
