## Task 2: `pdf-lib` export — inject translation annotation layer

**Goal:** `exportBilingualPdf(src: string): Promise<Blob>` — fetch original PDF bytes, walk pages, for each cached page inject a secondary text layer below each paragraph with the translation.

**Files:**
- Create `apps/extension/src/entrypoints/pdf-viewer/export/pdf-lib-writer.ts`
- Create `apps/extension/src/entrypoints/pdf-viewer/export/__tests__/pdf-lib-writer.test.ts`

**API:**

```ts
export interface ExportOptions {
  src: string
  fileHash: string
  targetLang: string
  providerId: string
}

export async function exportBilingualPdf(options: ExportOptions): Promise<Blob>
```

Internals:
1. Fetch PDF bytes
2. `PDFDocument.load(bytes)` via `pdf-lib`
3. For each page (0-indexed):
   a. `getCachedPage(fileHash, pageIndex, targetLang, providerId)` → paragraphs
   b. If no cache: skip (leave page as-is)
   c. For each paragraph: `page.drawText(translation, { x, y, size, font })` — coordinates translated from stored boundingBox (PDF units, no viewport scaling)
4. `pdfDoc.save()` → `Uint8Array` → `new Blob([bytes], { type: "application/pdf" })`

**Font handling:**
- If any paragraph contains CJK → `pdfDoc.embedFont(await fetch("/assets/fonts/noto-sans-cjk-sc-subset.otf").then(r => r.arrayBuffer()), { subset: true })`
- Else → standard `Helvetica` embedded

**Tests:**
- Mock `getCachedPage` + `fetch` for PDF bytes
- Assert `drawText` called for each paragraph with correct coordinates
- Assert CJK font embedded when CJK present
- Assert Latin-only uses Helvetica

**Commit:** `feat(pdf-viewer): export bilingual pdf via pdf-lib (M3 PR#C Task 2)`

## 风险 + 回退

| 风险 | 缓解 |
|------|------|
| CJK font asset too large | Subset to first 5000 common glyphs (~400KB); if still too big, split per-language (SC/TC/JA/KR) |
| pdf-lib can't handle coordinate systems from diverse PDFs | Use stored PDF-unit bounding boxes; skip pages where geometry is ambiguous |
| Export takes > 10s on 100-page PDFs | Show progress spinner; run in Web Worker if bottleneck (PR #C+1) |
| Watermark DOM hit by pdf.js re-render | Mount outside `.page` containers, at viewer-root level |
| Config-change invalidation races | `evictStaleConfigRows` runs on boot after fingerprint; single-shot, no mid-session invalidation |
