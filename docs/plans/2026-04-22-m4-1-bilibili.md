# M4.1 · Bilibili Adapter · 实施计划

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans.
> **Parent design:** `docs/plans/2026-04-22-m4-video-subtitles-design.md`
> **Predecessor:** M4.0 (#105) — PlatformRegistry abstraction

**Goal:** Add Bilibili subtitle translation overlay (first of 3 new platforms for M4). User opens a Bilibili video with CC → extension fetches subtitle JSON → overlays bilingual translation using the shared `UniversalVideoAdapter`.

**Architecture:**
- New `BilibiliSubtitlesFetcher` queries `api.bilibili.com/x/player/v2` via background `proxyFetch` (carries user cookies for logged-in access)
- `platforms/bilibili/config.ts` defines selectors + SPA navigation for the new player (`.bpx-player-*`)
- `platforms/bilibili/handler.ts` registers the platform with M4.0's `PlatformRegistry`
- `init-bilibili-subtitles.ts` handles SPA navigation via `history.pushState` monkey-patch (Bilibili doesn't emit clean `yt-navigate-*`-style events)
- `manifest.host_permissions` adds `*://*.bilibili.com/*` + `*://api.bilibili.com/*`

**API shape:**

```ts
export class BilibiliSubtitlesFetcher implements SubtitlesFetcher {
  kind: "bilibili" = "bilibili"

  async fetchTracks(videoId: { bvid?: string, aid?: string, cid: string }): Promise<RawTrack[]> {
    // Step 1: fetch player metadata via proxyFetch
    //   GET https://api.bilibili.com/x/player/v2?bvid={bvid}&cid={cid}
    // Step 2: extract data.subtitle.subtitles[] — each has { lan, lan_doc, subtitle_url }
    // Step 3: map to RawTrack[] with subtitle_url normalized (protocol-relative → https)
  }

  async fetchCues(track: RawTrack): Promise<SubtitleCue[]> {
    // GET <subtitle_url> via proxyFetch
    // Response: { body: [{ from, to, content, ... }, ...] }
    // Map `from`/`to` (seconds as float) to `startTime`/`endTime` (ms)
  }

  shouldUseSameTrack(prev, next) { /* cid-based equality */ }
}
```

**Key quirks to handle:**
- `cid` resolution: if only `bvid` is known, first call `api.bilibili.com/x/web-interface/view?bvid={bvid}` to get cid for the default part, then player/v2
- Multi-part videos (`?p=2`): read `?p=` query param, use corresponding `pages[p-1].cid`
- Unauthenticated access: Bilibili returns most CC subtitles without cookies; only special content is gated
- Subtitle URL is protocol-relative (`//aisubtitle.hdslb.com/...`) — prefix `https:` before fetch
- Rate-limit / Wbi signed endpoints: fall back to unsigned `x/player/v2` (simpler, works for basic subs)

## Task 3: Bilibili SPA navigation + init

**Files:**
- Create `apps/extension/src/entrypoints/subtitles.content/init-bilibili-subtitles.ts`

Unlike YouTube, Bilibili doesn't emit a stable custom navigation event that all pages fire. Use a hybrid approach:

1. Monkey-patch `history.pushState` + `history.replaceState` to emit a custom `bilibili:spa-navigated` event
2. Listen for `popstate` as a second trigger (back/forward)
3. Re-init adapter on any navigation where the BV changes

**Shape:**

```ts
export function initBilibiliSubtitles() {
  let mountedAdapter: ReturnType<typeof setupBilibiliSubtitles> | null = null
  let currentBvid: string | null = null

  const tryInit = async () => {
    const bvid = getCurrentBvid()
    if (!bvid) return
    if (bvid === currentBvid) return   // same video, skip
    currentBvid = bvid

    if (!mountedAdapter) {
      mountedAdapter = setupBilibiliSubtitles()
    }
    await mountSubtitlesUI({ adapter: mountedAdapter, config: bilibiliConfig })
    void mountedAdapter.initialize()
  }

  patchHistoryForSpaNavigation()  // monkey-patch once
  window.addEventListener("bilibili:spa-navigated", () => void tryInit())
  window.addEventListener("popstate", () => void tryInit())

  void tryInit()   // initial page load
}
```

Register the Bilibili handler:
```ts
// in runtime.ts
registry.register(bilibiliHandler)  // one-line addition
```

## Risks + Mitigations

| Risk | Mitigation |
|------|------------|
| Wbi signed endpoints required in the future | Keep `x/player/v2` (unsigned) as default; add sign helper only if Bilibili returns 4xx |
| Legacy player on some pages | Skip — URL pattern matches `/video/BV` only; legacy live pages excluded |
| Cookies required for logged-in CC | `proxyFetch` already carries cookies for matched domains (M1/M0 pattern) |
| CSS conflicts with Bilibili player | Shadow Root isolates our overlay; position absolute inside `.bpx-player-container` |
| Bilibili danmaku layer overlaps | `z-index: 9999` on our overlay container; danmaku is lower |
